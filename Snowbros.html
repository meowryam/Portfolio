Project Overview

What it is & why it was built.

Snow Bros is a remake of the 1990 arcade platformer originally developed by Toaplan. Players control one or two characters who throw snowballs to freeze enemies and kick them across the stage, creating chain reactions to clear each level.

Built in C++ using SFML, the project focused on recreating the core gameplay while emphasizing clean object-oriented design. Rather than prioritizing quick implementation, the architecture was designed to separate entities, animation, collision handling, and game states into reusable systems.

The project served as an exercise in building maintainable game code, demonstrating how principles such as inheritance, polymorphism, and separation of concerns can be applied to create a structured and extensible codebase.

Features

What's inside the game.

🎮

Two-Player Co-op

Local two-player mode with separate input handling for each character. Both players can independently throw snowballs, move, and jump across all platforms.

❄️

Snowball Mechanics

Projectiles are fired left or right based on player facing direction. Enemies hit by snowballs enter a "frozen" state and can then be kicked to defeat nearby enemies in chain reactions.

🏃

Sprite Animation System

Frame-based animations driven by delta time, not frame rate. Each entity has distinct idle, run, jump, attack, and death animation states stored in sprite sheets and switched via state flags.

🧱

Tile-Based Collision

Levels are built from tile maps. Collision is resolved per-tile, allowing characters to stand on platforms, fall through gaps, and correctly handle edge cases at tile boundaries.

🤖

Enemy AI

Enemies use a simple finite state machine — patrolling, chasing, fleeing when encased in snow. Physics-driven movement ensures enemies realistically fall off platforms.

🎯

Score & Level Progression

Points awarded per enemy defeated and per chain reaction. Levels clear when all enemies are eliminated, then load the next stage with increased difficulty.
